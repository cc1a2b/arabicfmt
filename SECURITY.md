# Security Policy

## Supported versions

The latest published version on npm receives security fixes. arabicfmt has
**zero runtime dependencies**, so the supply-chain surface for consumers is
minimal.

| Version | Supported |
|---------|-----------|
| latest (0.1.x) | ✅ |
| older          | ❌ |

## Reporting a vulnerability

Please **do not** open a public issue for security problems.

- Preferred: use GitHub's **private vulnerability reporting** — go to the
  repository's **Security** tab → **Report a vulnerability**.
- Alternatively, email the maintainer at the address listed in
  [`package.json`](./package.json) (`author` field).

Please include: affected version, a description, and a minimal reproduction.
You'll get an acknowledgement as soon as possible, and a fix or mitigation
will be released promptly once confirmed.

Thank you for helping keep arabicfmt and its users safe.
